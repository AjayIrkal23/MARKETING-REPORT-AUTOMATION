## Things We Tried That Failed

- **[2026-05-31]** **Large parallel-builder workflows can emit a duplicate, mis-named tree.** The 39-agent JSW Stock build produced a complete orphan duplicate of the frontend AND backend under a hallucinated `jvml`/`jvml_stock` naming (e.g. `useJvmlStockConfig.ts`, `app/services/jvml_stock/`), parallel to the correct `jsw`/`jsw_stock` files. It passed `tsc -b` (the orphan tree is internally consistent and unreferenced from the entry graph) so only **eslint** (which lints every file) caught it. **Lesson:** after a big multi-agent build, grep for unexpected sibling names (`grep -rl <typo-variant> src/`) and run eslint, not just tsc. The orphan trees were removed; the wired tree (`jsw`/`jsw_stock`, imported by `App.tsx`/`routes/__init__.py`) is the canonical one.
- **[2026-05-31]** **The malformed `"1.057.000"` cell is in `tensile_strength_mpa_b` (col BL), NOT `Width (mm)`** ‚ÄĒ `macro_docs/zsd-currstk-hr.md` and the original SPEC were wrong. Live parse confirmed `Width (mm)` is entirely shared-string-backed (all 17,324 values are `str`). At the raw-parse layer the malformed value stays a `str`; because `tensile_strength_mpa_b` is a `number` column, `coerce_value` returns `None` for it. No special-casing needed.
- **[2026-05-31]** A Beanie 2.1.0 `Document` (e.g. `JswStock(...)`) **cannot be instantiated before `init_beanie`** ‚ÄĒ raises `CollectionWasNotInitialized`. To smoke-test ingestion/coercion without a server, either run `init_beanie` against a live Mongo first, or validate `coerce_value` outputs directly without constructing the Document.
- **[2026-05-30]** Beanie 2.1.0: `AuditLog.find(filt).distinct(field)` raises `AttributeError: 'FindMany' object has no attribute 'distinct'`. In Beanie 2.x `distinct` is **not** a `FindMany` method ‚ÄĒ it is a `Document` classmethod: `await AuditLog.distinct(key, filter)` (coroutine ‚Üí `list`, proxies pymongo `collection.distinct`). `FindMany` only terminates with `.to_list()`, `.count()`, `.aggregate()`. Fixed both the audit `options`/`facets` services this way. (Source-driven check: `dir(FindMany)` vs `dir(Document)` in the installed venv.)

## Known Fragile Areas

- **`macro_files/ZSD_CURRSTK_HR.xlsx`** ‚ÄĒ Contains invalid XML (cell `"1.057.000"`). `openpyxl` will crash on load. See "Things We Tried" above for the fix procedure. - **Customer code join key** ‚ÄĒ SAP Party Code in `ZSD_CURRSTK_HR.xlsx` is zero-padded to 10 digits. Strip leading zeros before joining to the 8-digit customer codes in the other two files. Short codes `8451‚Äď8499` / `8001`-style are internal JSW stock-transfer yards ‚ÄĒ not external customers.

When cloning `jsw_*`‚Üí`jvml_*` with a blind `s/jsw/jvml/` sed, `jsw_grade`‚Üí`jvml_grade` is WRONG (the field/column stays `jsw_grade` / "JSW Grade" in the JVML domain too). - **`macro_files/JVML Stock (99).xlsx`** ‚ÄĒ The JVML stock export: same 72-column SAP structure as `ZSD_CURRSTK_HR.xlsx` (3388 rows; same `Tensile Strength MPa (B)` malformed-numeric column; chemistry in scientific notation; `Production Date`/`CP End Date` as Excel serials; `LC Exp Date` text).
