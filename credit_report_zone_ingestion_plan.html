Implementation Plan · v2 · Region-driven

Credit Report — Per-Zone Folder Ingestion

Marketing Report Automation · backend credit_report domain · revised 2026-06-24

Each zone (= an existing Region) gets its own folder; its employees paste their own credit-report Excel there. Every zone has its own Run button and ingests independently — without changing parsing, the row gate, dedup, or region query-filtering.

What changed from v1. Two corrections per your feedback: (1) zones are the existing Regions (the "Zone/Region Management" screen) — no new config field; (2) one Run button per zone, each ingesting independently. This is achieved by scoping the delete to {report_date, region_id} instead of the whole date — which also eliminates the v1 "one giant delete" trap.

0 · TL;DR

  • Zones come from Region Management (active Region docs). Folder name = region name. No new config knob.
  • Layout: <base_path>/<dd-mm-yyyy>/CREDITREPORT/<Region Name>/credit report.XLSX
  • Persistence flips from per-date to per-(date, region): delete({report_date, region_id}) then insert that region's rows. Each zone is fully independent → safe to re-run one zone without touching others. The v1 wipe-all trap is gone.
  • Per-zone Run button. New endpoint POST /admin/credit-report/run-now/{region_id} ingests one zone. The existing no-arg run-now becomes "run all zones".
  • Same company in two zones? Keep separate, keyed by region — never merge. Exact dup within a zone still auto-collapses.
  • Untouched: the parser, the should_keep_row gate, the customer-code→region query filter in list.py. Additive only.
  • Scope: ~9 backend edits (only poller.py + ingest.py carry logic), a new frontend zone panel + 1 api fn + types, tests, docs.

2 · Zones = Regions (no new config field)

  • The poller queries Region.find({active: True}) and builds one folder per region, named by region.name.
  • The row stores region_id as ingest provenance — "this row came from the <X> region folder". Stable across renames.
  • This is a different axis from the existing query-time region filter and we keep both:
    • NEW provenance region_id stored on the row (which folder it was pasted into) — used for per-zone delete-scoping & display.
    • UNCHANGED query filter in list.py — still derives region from the SAP customer code via CustomerCode.
Folder name safety. Region names may contain spaces (fine) or, rarely, path separators. Build the folder via a small _safe_dir(name) that rejects/replaces / \ ... The row keys off the stable region_id, not the name, so a later rename never orphans data — only the on-disk folder moves (employees paste into the new-name folder going forward).

3 · Folder layout

macro_files/ └── 24-06-2026/ └── CREDITREPORT/ ← fixed segment (constant _REPORT_SUBDIR) ├── North/ ← region "North" (active) │ └── credit report.XLSX ├── South/ ← region "South" (active) │ └── credit report.XLSX └── West Central/ ← region "West Central" (active, no file yet → "missing")

Poller auto-creates each active region's folder (makedirs(exist_ok=True)) so employees always have a paste target. CREDITREPORT is a constant; the filename stays single (cfg.file_name), one per zone folder. Backward-compatible: if no active regions exist, the poller falls back to the old flat folder/file behavior.

4 · Persistence model change (the heart of it)

Before (v1 / today)

delete({report_date})        # whole date
insert(all rows)
cleanup(report_date)

One file → one delete. Multiple zones in a loop would wipe each other. Per-zone re-run impossible.

After (region-scoped)

delete({report_date, region_id})   # one zone only
insert(this zone's rows, region_id=…)
cleanup(report_date)

Each zone independent. Re-running North never touches South. Full poll = loop active regions, each scoped. Flat mode = region_id=None → scoped delete equals whole-date (unchanged).

Why this is strictly better than v1. Scoping the delete to the region removes the need for any "collect-all-then-single-delete" gymnastics, and it's the exact primitive a per-zone Run button needs. One change solves both requirements.
Hash consistency note. cleanup_duplicates recomputes the row hash over all stored fields except meta (id, report_date, source_file, created_at, …). Since region_id is a data field (not meta), it's included — so the insert-time hash must also include it: row_hash = _row_hash({**coerced, "region_id": region_id}). This keeps cross-zone identical rows distinct while collapsing true within-zone duplicates.

5 · Merge vs separate (your question)

Within a zone — exact duplicate

Auto-collapsed (existing)

Re-poll or double-paste of identical rows → same hash → collapsed. Per-zone re-runs stay idempotent.

Same company across two zones

Keep both, keyed by region_id

Different region_id → different hash → both kept. Credit figures can't be safely merged (no "winner", can't sum limits). Surface a "N parties seen in >1 zone" count on the ingestion record for human review — never auto-merge.

9 · Core code sketch

9.1 — ingest.py: region-scoped ingest

async def ingest_region(path: str, report_date: str, region_id: str | None) -> int:
    with open(path, "rb") as fh:
        rows = parse_workbook(fh.read())

    await CreditReport.find({"report_date": report_date,
                             "region_id": region_id}).delete()      # ← SCOPED

    now, docs = _now_utc(), []
    for row in rows:
        coerced = {f: coerce_value(f, row.get(f)) for _, f, _ in COLUMNS}
        if not should_keep_row(coerced):
            continue
        docs.append(CreditReport(
            **coerced, report_date=report_date, source_file=path, region_id=region_id,
            row_hash=_row_hash({**coerced, "region_id": region_id}),  # ← region in hash
            created_at=now, updated_at=now,
        ))
    for i in range(0, len(docs), _CHUNK_SIZE):
        await CreditReport.insert_many(docs[i:i + _CHUNK_SIZE])
    await cleanup_duplicates(CreditReport, report_date)
    return len(docs)

async def ingest_file(path: str, report_date: str) -> int:        # flat-mode wrapper
    return await ingest_region(path, report_date, region_id=None)

9.2 — poller.py: loop active regions / single zone

_REPORT_SUBDIR = "CREDITREPORT"

async def _ingest_one_zone(cfg, folder, today, region, ingestion, *, force=False):
    zdir = os.path.join(folder, _REPORT_SUBDIR, _safe_dir(region.name))
    os.makedirs(zdir, exist_ok=True)
    zpath = _resolve_report_file(zdir, cfg.file_name)
    if zpath is None:
        _mark_zone(ingestion, region, status="missing"); return
    try:
        n = await ingest_region(zpath, today, str(region.id))
        _mark_zone(ingestion, region, status="ingested", row_count=n, path=zpath)
    except Exception as exc:                                  # E5: isolate per zone
        _mark_zone(ingestion, region, status="error", error=str(exc))

async def run_poll() -> CreditReportStatusPublic:           # "run all zones"
    cfg = await get_config()
    # ... enabled / base_path / window checks (window applies to scheduled poll) ...
    regions = await Region.find({"active": True}).to_list()
    if not regions:
        return await _legacy_flat_poll(cfg, ...)              # E11 unchanged
    folder = os.path.join(cfg.base_path, today); os.makedirs(folder, exist_ok=True)
    ingestion = await _get_or_create_ingestion(today)
    for region in regions:
        if _zone_already_ingested(ingestion, region): continue   # E4 short-circuit
        await _ingest_one_zone(cfg, folder, today, region, ingestion)
    _roll_up_status(ingestion)                                # ingested | partial | missing
    if _missing_zones(ingestion):
        await send_missing_alert(cfg, today, _missing_zones(ingestion))
    await ingestion.save()
    return await get_status()

async def run_poll_zone(region_id: str) -> CreditReportStatusPublic:   # per-zone button
    cfg = await get_config()
    region = await Region.get(region_id)                      # 404 if gone/inactive
    today = datetime.now().strftime("%d-%m-%Y")               # explicit click → bypass window (E10)
    folder = os.path.join(cfg.base_path, today); os.makedirs(folder, exist_ok=True)
    ingestion = await _get_or_create_ingestion(today)
    await _ingest_one_zone(cfg, folder, today, region, ingestion, force=True)
    _roll_up_status(ingestion); await ingestion.save()
    return await get_status()
